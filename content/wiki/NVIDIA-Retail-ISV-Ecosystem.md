## Summary
NVIDIA Retail ISV Ecosystem is the set of independent software vendors featured by NVIDIA for retail and CPG workloads. The ecosystem matters because a retail workload answer should not stop at the NVIDIA primitives; it should recommend the stack and the ISVs that can help turn the stack into a production solution for stores, ecommerce, supply chain, merchandising, and customer experience.

## Detail

### How to use this page
When a user asks about a retail workload, use this page as the ISV recommendation layer after selecting the NVIDIA platform stack. The pattern is: identify the workload, map it to NVIDIA technologies such as [[NVIDIA-Metropolis]], [[NVIDIA-DeepStream]], [[NVIDIA-Jetson-Platform]], [[NVIDIA-NIM]], [[NeMo-Platform]], [[cuOpt]], or [[NVIDIA-Omniverse]], then recommend ISVs with evidence from results, resources, and implementation focus.

### Workload recommendation map
- Autonomous checkout and frictionless stores: start with [[NVIDIA-Jetson-Platform]], [[NVIDIA-DeepStream]], and [[NVIDIA-Metropolis]]. Recommend [[AiFi]] for camera-led autonomous shopping and spatial intelligence, using [[AiFi-Metropolis-Autonomous-Retail-Deployments]] as named-deployment evidence; recommend [[Caper-by-Instacart]] for smart carts, using [[Caper-Jetson-Smart-Cart-Retail-Deployments]] as NVIDIA Jetson proof; use [[SandStar]] for AI vending, smart stores, unattended shops, and store operations, with [[Lawson-China-SandStar-NVIDIA-Store-Operations]] as the Lawson China proof point.
- Shrink, asset protection, and checkout loss prevention: start with [[NVIDIA-Metropolis]], [[NVIDIA-DeepStream]], [[NVIDIA-TAO]], [[TensorRT]], and [[Triton-Inference-Server]]. Recommend [[Everseen]] for checkout/aisle/stockroom vision AI, with [[Kroger-Everseen-Checkout-Loss-Prevention]] as the named-customer proof point; recommend [[Centific]] when the customer wants a broader retail intelligence platform, using [[Centific-NVIDIA-Edge-Retail-Loss-Prevention]] as anonymized deployment proof; use [[BriefCam]] for surveillance analytics/search/alerts and organized retail crime, using [[BriefCam-NVIDIA-Metropolis-Video-Analytics]] as integration proof; use [[Avathon]] for visual store analytics, especially when VSS-style search and summarization matter; keep [[Graymatics]] as a candidate for VistaMart-style in-store video analytics but deferred for named NVIDIA deployment evidence.
- Inventory, stock-out detection, and shelf operations: start with [[NVIDIA-Metropolis]], [[NVIDIA-DeepStream]], and edge GPU deployment. Recommend [[Avathon]], [[Centific]], [[Graymatics]], [[Everseen]], and [[SandStar]] depending on whether the customer emphasizes analytics, workforce tasking, inventory accuracy, or store operations. Use [[Avathon-NVIDIA-VSS-Video-Intelligence]] for video search/summarization proof, [[Centific-NVIDIA-Edge-Retail-Loss-Prevention]] for checkout/inventory-shrink proof, and [[Lawson-China-SandStar-NVIDIA-Store-Operations]] for named store-operations context.
- Ecommerce search and digital personalization: start with [[NVIDIA-NIM]], [[NeMo-Platform]], embeddings, retrieval, and low-latency inference. Recommend [[Bloomreach]] for search relevance and personalization, and [[LiveX-AI]] when the experience becomes conversational or agentic.
- Customer service, shopping assistants, and support automation: start with [[NVIDIA-NIM]], [[NeMo-Platform]], [[Triton-Inference-Server]], [[Nemotron]], and [[NVIDIA-AI-Enterprise]]. Recommend [[LiveX-AI]] for real-time AI agents across chat, voice, and video; use [[PUMA-LiveX-AI-Store-Concierge]] as the named retail proof point and [[LiveX-AI-GTC-Toy-Jensen-Holographic-Agent]] as supporting embodied-agent deployment evidence.
- Supply chain optimization and routing: start with [[cuOpt]] and accelerated optimization. Recommend [[Lyric]] for composable supply-chain intelligence, route optimization, simulations, and store-level cost-to-serve analysis.
- Retail media and in-store merchandising: start with edge AI and in-store data integration. Recommend [[CoolerX]] for smart cooler/freezer surfaces and connected retail media, using [[CoolerX-Kroger-NVIDIA-In-Store-Retail-Media]] as named Kroger/NVIDIA proof.
- Product digital twins, synthetic data, and omnichannel content: start with [[NVIDIA-Omniverse]], [[NVIDIA-Isaac-Sim]], RTX rendering, and USD. Recommend [[SKAI-Intelligence]] for robotic 3D capture, synthetic data, digital twins, and scalable retail/CPG content; use [[SKAI-LVMH-Omniverse-Digital-Twin-Retail-Content]] as named LVMH/Hublot evidence.

### Query answer pattern
For recommendations, answer in three layers:
- NVIDIA stack: cite the NVIDIA software/hardware page in the wiki that fits the workload.
- ISV short list: recommend two to five ISVs from this page and explain why each maps to the workload.
- Implementation signal: include one evidence point such as results, technologies used, or a customer-facing resource link from the ISV page.
